# 微博热搜产品创意分析

## 概述

本技能用于自动抓取微博热搜榜单，对每个热点话题进行深度分析，提取可落地的产品创意，并生成美观的 HTML 分析报告。

**核心流程**：抓取热搜 -> 搜索背景 -> AI分析 -> 生成报告

## 工作流

### 阶段 1：抓取微博热搜榜单

使用天行数据 API 接口获取微博热搜数据。

**API 地址**：`https://apis.tianapi.com/weibohot/index?key=c5ecfa20b2589e370788ee0f1e94c3db`

**API 响应结构**（天行数据 weibohot 接口）：

```json
{
  "code": 200,
  "msg": "success",
  "result": {
    "update_time": "2025-01-01 12:00:00",
    "data": [
      {
        "word": "热搜关键词",
        "hot_value": "1234567",
        "label_name": "热/新/沸/爆",
        "category": "分类"
      }
    ]
  }
}
```

**执行步骤**：

1. 使用 Bash 工具通过 `curl` 请求 API：

```bash
curl -s "https://apis.tianapi.com/weibohot/index?key=c5ecfa20b2589e370788ee0f1e94c3db" | python3 -c "
import sys, json
data = json.load(sys.stdin)
if data.get('code') != 200:
    print('ERROR: API returned code ' + str(data.get('code')) + ', msg: ' + data.get('msg', 'unknown'))
    sys.exit(1)
items = data.get('result', {}).get('data', [])
if not items:
    print('ERROR: No data returned from API')
    sys.exit(1)
for item in items[:30]:
    word = item.get('word', '')
    hot = item.get('hot_value', '0')
    label = item.get('label_name', '')
    category = item.get('category', '')
    print(json.dumps({'word': word, 'hot_value': hot, 'label': label, 'category': category}, ensure_ascii=False))
"
```

2. 将解析后的热搜数据保存为 JSON 文件 `hotspot_data.json`
3. 如果 API 请求失败（如 code 非 200），提示错误信息并建议稍后重试

### 阶段 2：搜索每个热点的背景信息

对热搜榜单中的 **前 15 个** 热点话题逐一搜索（避免过多请求）。

**对每个热点执行**：

1. 使用 WebSearch 工具搜索：`"{热点关键词} 事件 起因 经过"`
2. 如果搜索结果不够充分，追加搜索：`"{热点关键词} 最新消息 进展"`
3. 综合搜索结果，为每个热点整理以下信息：
   - **事件概述**：一句话概括（30字以内）
   - **事件脉络**：按时间线梳理关键节点（3-5个节点）
   - **核心矛盾/看点**：为什么这件事会引发关注
   - **涉及领域**：如科技、娱乐、社会、经济、教育等
   - **情感倾向**：正面/负面/中性/争议

4. 将所有背景信息追加到 `hotspot_data.json` 中对应条目

**注意**：
- 每次搜索 3-5 个热点（并行 WebSearch 调用），分批完成
- 如果某个热点搜索不到有效信息，标注为"信息不足"并跳过深度分析

### 阶段 3：AI 产品创意分析

对每个有充分背景信息的热点进行产品创意分析。

**评分体系**（总分 100 分）：

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 有趣度 | 80分 | 话题新颖性(30)、大众共鸣度(25)、传播潜力(25) |
| 有用度 | 20分 | 实际需求强度(10)、可落地程度(10) |

**对每个热点输出**：

1. **综合评分**：总分（0-100），精确到整数
2. **评分细项**：有趣度得分、有用度得分
3. **产品创意**（至少1个，最多3个）：
   - **产品名称**：简洁有力，最好4-8个字
   - **Slogan**：一句话描述产品价值（15字以内）
   - **核心功能**：3-5个核心功能点，每个功能用一句话说明
   - **目标用户**：具体描述用户画像（年龄、职业、场景）
   - **差异化亮点**：相比现有产品有什么独特之处
   - **商业变现思路**：简要说明可能的盈利模式
4. **分析理由**：说明为什么从这个热点得出了这样的产品创意

### 阶段 4：生成 HTML 报告

读取 [html-template.md](html-template.md) 获取完整的 HTML 模板，然后将分析结果填入模板，生成最终 HTML 报告文件。

**报告命名**：`weibo_hotspot_report_YYYYMMDD_HHmm.html`（使用当前日期时间）

**报告核心要求**：
- 列表式布局，按评分从高到低排序
- 每个热点卡片包含：事件脉络 + 产品创意详情 + 综合评分
- 评分视觉区分：
  - **优秀（>80分）**：金色/橙色高亮边框 + 火焰图标，置顶展示
  - **良好（60-80分）**：蓝色边框 + 星标
  - **一般（<60分）**：灰色边框，标准展示
- 页面顶部有统计摘要：总数、优秀/良好/一般数量、平均分
- 响应式设计，支持移动端查看
- 支持按评分/领域筛选

**生成步骤**：

1. 读取 `html-template.md` 模板文件
2. 将所有热点分析数据（来自 `hotspot_data.json`）填入模板
3. 按综合评分降序排列
4. 写入 HTML 文件到项目根目录
5. 使用 Bash 工具在浏览器中打开（macOS 使用 `open` 命令）

```
- 平均得分：XX 分

🔥 TOP 3 产品创意：
1. [热点A] -> 产品名：XXX | 评分：XX
2. [热点B] -> 产品名：XXX | 评分：XX
3. [热点C] -> 产品名：XXX | 评分：XX

```

### 错误处理

- **API 不可达**：提示天行数据 API 请求失败，显示错误码和信息，建议稍后重试
- **搜索限流**：跳过当前热点，继续分析下一个，最终报告标注跳过原因
- **无有效热点**：提示 API 返回数据为空，建议检查接口或稍后重试

## 使用示例

```
# 直接调用，使用内置 API 地址
/weibo-hotspot
```

## 注意事项

- 默认只分析热搜 **前 15 个** 话题以保证分析质量
- WebSearch 调用分批进行，每批 3-5 个热点
- HTML 报告是独立文件，不依赖外部资源（CSS/JS 内联）
- 所有数据临时存储在项目目录，分析完成后保留 JSON 数据文件供后续查阅
- API 使用天行数据（tianapi.com）的微博热搜接口，密钥已内置
